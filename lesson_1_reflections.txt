查看文件的两个版本之间的差异如何帮助你了解引入的错误？
首先，比较最新版和最后能正常运行版本之间差别，可以知道新版本都有那些新内容。
其次，比较不同点能发现新版本的拼写和格式错误；
最后，综合比较不同点能够发现逻辑错误。
从长远来看，能轻松访问文件的整个历史记录将如何提高编程者的效率？
1. 不用想着怎样命名；
2. 不需要记忆文件储存位置；
3. 可以直接在源文件上更改。
你认为，手动选择何时创建提交（像在 Git 中做的那样）与自动保存版本（如 Google Docs 所做的那样）各有何优缺点？
手动保存提交更适合于逻辑性任务，一段工作结束之后整理提交，但是常常会因为意外情况导致文件丢失。
自动保存提交更适合与工程性任务，思路连续尽可能把每一步保存下来，防止遗失。
你认为为什么一些版本控制系统（例如 Git）允许在一次提交中保存多个文件，而其他系统（例如 Google Docs）单独对待每个文件？
不同版本控制系统的服务对象是不同的，大部分任务具有独立性，可以在一个文件中完成据大部分内容。从经济上考虑，单独对待每一个文件更有优势。
如何使用命令 git log 和 git diff 来查看文件的历史记录？
需要在仓库目录下使用，按下键向下浏览，同时结束查看需要按“Q”。git diff:与FC和diff相似，后面接两个不同版本的ID即可比较两个不同版本之间的区别。
使用版本控制，如何让你更有信心进行有可能会造成破坏的更改？
可以回到以前的版本，即使修改时误删、误改，仍然能够找到最近可运行版本。
